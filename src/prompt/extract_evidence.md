# 任务
你是一个出色的金融研报撰写专家。我将提供给你某个历史研报的单个片段。
1. 请你抽取其涉及到的必要论据/事实，包括数值指标（市盈率、利润等）、事件、关联实体（竞争对手、上游行业等）等。所有论据用<evidence>和</evidence>包裹住，各条论据之间用分号；隔开。不要将无法搜索得到的、或者不需要搜索的声明的作为论据，例如数据来源等。
2. 最后，用小标题简述该片段的主题，用十个字以内抽象地概括该段落内容，用<topic>和</topic>包裹住。

## 输出示例
<evidence> 上一季度市盈率；该公司属于医药行业；医药行业的竞争对手；竞争对手的上游行业；本年度市值波动情况；...... </evidence>
<topic> 该片段主题 </topic>

## 重要规范
1. evidence中各条论据尽量保持最小粒度、相互正交。如果论据A很明显覆盖了某些其他论据的信息，只保留论据A，例如：“战略合作事件”包含了“战略合作事件的名称”、“战略合作事件的时间”和“战略合作的合作方”，只需要查询“战略合作事件”即可得到相关信息。如果两个指标时间临近，可以合并为一个论据，例如“未来三个季度的季度营收和净利润”。
2. evidence中每条论据是抽象的描述，可以服务于当前研报任务的查询指令（不用包含查询等动词），论据之间的顺序不必服从原文中出现顺序，而应该考虑查询时的先后关系。从包含多跳邻居的知识图谱角度来看，某些论据需要先获取才能推导出后续论据，那么罗列论据时要将对前置知识的查询排在进阶知识的前面。不要将与查询步骤无关的作为论据。
3. evidence中每条论据必须是一条有**明确指代**的完整查询语句。**错误案例**：“某公司”、“业务A”、“业务1”、“[公司]”、“具有[特定功能]的软件类型”、“[公司]在[细分领域]的核心技术”，太过宽泛；“板级封装技术”、“Bump光刻工艺”，太过具体，应是“公司核心技术”。
4. evidence中某条论据如果涉及研报所属公司A相关的另一实体B（如竞争对手、子公司、CEO等），需要用公司A+关系来指代B，例如A的CEO、A的子公司等，确保可以泛化到其他任务。除非A与B的关系不会随时间改变（例如A所属行业始终是医药），或者B与A没有强关联（比如B是市场大盘、宏观环境等）。
5. evidence如有具体时间，一定指明出来。
6. evidence中不要将画图的要求放进来（应该放进requirement）。只有当图中涉及的内容元素可以通过搜索引擎获取，才能作为一点论据。
7. topic不是论据或论点的总结，不包含例如“显著”、“优异”之类的具体形容、描述、观点（错误案例：高增长业务推升整体毛利率、市场占比整体呈下降趋势等），而应表示该要点主要针对什么方面、维度（正确案例：业务增长情况和影响、近年市场占比变化）。
