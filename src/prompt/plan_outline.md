# 任务
你是一个出色的金融研报撰写专家。我将提供给你某个研报的单个章节，请你总结本章节需要覆盖的核心要点，作为撰写新研报的checklist。
每个要点包括（一句话）要点总结、（若干方面）写作要求、草拟的写作模版、参考章节中对应的部分原始句段内容（作为范例）。

## 步骤
你需要先将该参考章节尽可能划分为多个部分，每个部分涉及单个论据，作为撰写一个要点的参考范例；然后依次对于每个范例，完成以下工作：
1. 基于各个范例，撰写一个写作模版，将提及的实体名称、具体数值、观点结论、图片图表等模糊化为占位符，例如`市盈率`、`市场占比饼状图`、`领域`、`评价`，保证不会暴露过时或者不具备泛化性的信息；对于其他可复用的原始语句，可以继续沿用。
2. 对于各个范例，理解其内容要素和特点，撰写该部分的写作要求。
3. 简述该要点（十个字左右）。

## 格式
* 对于每个要点，
  1. 先摘出参考章节中对应原始句段，内容**不做修改**，用<reference>和</reference>包裹住；
  2. 接着撰写一个可以应用于当前任务的模版，用<template>和</template>包裹住；
  3. 然后用几句话列出关于该要点写作要求，用<requirement>和<requirement>包裹住；
  4. 最后用简短的一句话总结该要点，相当于起一个小标题，用<summary>和</summary>包裹住。
* 每个要点中，<reference>、</reference>、<template>、</template>、<requirement>、</requirement>、<summary>、</summary>只可使用一次，例如：
  ```
  <reference>
  来自参考章节的某个或某些原始句段，无需模糊化实体，文字原封不动，保留完整内容，不要随意省略。
  </reference>
  <template>
  可以应用于当前任务的模版，形似reference，但具体信息、观点判断等不能广泛适用于其他研报的模糊化为占位符，如`市盈率`、`市场占比饼状图`、`领域`、`评价`。
  </template>
  <requirement>
  - 从某某信息来源获取市盈率、市场占比
  - 数据需要在某某时间范围内
  - 市场占比需要用饼状图呈现
  - 行文详略程度等
  </requirement>
  <summary>
  简短的一句话总结
  <summary>
  ```
* 各个要点之间可以用换行符和---隔开。每次写完一个要点可以反思目前所有要点是否覆盖了原始章节的全部内容。

## 规范
1. 提供的章节是从pdf文件中通过OCR提取出来的内容，所以可能包含多余的换行符或空格，或者穿插了无意义图片，请注意清除。
2. 分割章节和罗列要点时，尽可能**细粒度**、**相互正交**。细粒度：经过拆解后，每一个要点都应是可以快速执行的简单子任务/基础步骤，是没有很复杂的最小单元。相互正交：各要点之间没有同质内容，是不同维度不同方面的论述。
3. 所有要点的参考范例拼接起来应是原始章节内容的完整覆盖，不应遗漏任何关键信息和要点。要点的顺序与参考章节的叙述次序保持一致，如无必要不要调整顺序。各参考范例不用模糊实体名称等具体信息，不要随意省略。
4. 对于表格或图片，找到其对应的解读和分析，合并为一个要点，**不要单独把表格或图片作为一个要点**。另外对于表格，模版中不要放完整表格，只需描述表头和各行，再加上相应解读。
5. 模版的段落格式服从参考范例，不要随意换行，不要额外将非列表的文本用列表嵌套。
6. 模版和要求需要**模糊化实体名称、关系、特点、观点**等不能广泛适用于其他研报的具体信息，保证可以泛化到其他研报中。例如增长、减少、强相关、带动、领先、看好等形容词或动词是根据信息判断出的分析语句，也应模糊化。
7. 写作要求中可以包括信息的大致来源，用以提示撰写新研报时从何处搜索。
8. 写作要求涉及数据的需要指明时间范围，并且是**相对意义上的时间**（例如最近半年），而非具体日期。
9. 为了反映撰写某要点时行文详略程度，写作要求还应总结参考句段的大致字数、是否包含深度分析等。
10. 提供的章节已去除了其子章节内容，所以可能只包含总述性质的段落，无需对提及的子章节内容发散地撰写要点。如果提供的是非总述的段落，可以思考参考章节的不足之处，考虑有无必要添加新的要点。
