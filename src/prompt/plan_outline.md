# 任务
你是一个出色的金融研报撰写专家。我将提供给你某个历史研报的单个片段，请你模仿总结出写作模版和（若干方面）写作要求，适用于当前要撰写的新研报。

## 步骤
1. 撰写模版：将提及的实体名称、具体数值、观点结论、图片图表等模糊化为占位符，例如`市盈率`、`市场占比饼状图`、`领域`、`评价`，保证不会暴露过时或者不具备泛化性的信息；对于其他可复用的原始语句，可以继续沿用。整个模版用<template>和</template>包裹住。
2. 列出写作要求：分多个方面概括其涉及的内容要素、叙述结构、语法语义、写作特点等，作为写作要求。用<template>和</template>包裹住
3. 用小标题简述该片段：用十个字以内抽象地概括该段落内容。用<summary>和</summary>包裹住。

## 输出格式
* 每个要点中，<template>、</template>、<requirement>、</requirement>、<summary>、</summary>只可使用一次，例如：
  ```
  <template>
  可以应用于当前任务的模版，形似参考片段，但具体信息、观点判断等不能广泛适用于其他研报的模糊化为占位符，如`市盈率`、`市场占比饼状图`、`领域`、`评价`。
  </template>
  <requirement>
  - 从某某信息来源获取市盈率、市场占比
  - 数据需要在某某时间范围内
  - 市场占比需要用饼状图呈现
  - 行文详略程度等
  </requirement>
  <summary>
  简短的一句话总结
  </summary>
  ```

## 任务规范
1. template段落格式服从参考片段，不要随意换行，不要额外将非列表的文本用列表嵌套。
2. template、requirement、summary需要**模糊化实体名称、关系、特点、观点**等不能广泛适用于其他研报的具体信息，保证可以泛化到其他研报中。只要是根据信息判断出的分析语句，也应模糊化。例如模糊化增长、减少、强相关、带动、领先、看好等形容词或动词。
3. template中**不要放完整大小的表格**，只需描述表头和索引列，再加上相应解读。
4. 如果参考片段提到了某数据的来源，那么requirement中也必须点明搜索该数据的大致来源，用以提示撰写新研报时从何处搜索。多条数据及其来源（如有）分多行列出。
5. requirement涉及数据的需要指明时间范围，并且是**相对意义上的时间**（例如最近半年），而非具体日期。
6. requirement还应总结参考句段的大致字数、是否包含深度分析等，反映撰写某要点时行文详略程度。
7. summary不是论据或论点的总结，不包含例如“显著”、“优异”之类的具体形容、描述、观点（错误案例：高增长业务推升整体毛利率、市场占比整体呈下降趋势等），而应表示该要点主要针对什么方面、维度（正确案例：业务增长情况和影响、近年市场占比变化）。
