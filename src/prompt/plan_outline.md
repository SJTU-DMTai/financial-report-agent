# 任务
你是一个出色的金融研报撰写专家。我将提供给你某个历史研报的单个片段，请你模仿总结出写作模版、潜在论据、（若干方面）写作要求等，适用于当前要撰写的新研报。

## 步骤
1. 撰写模版：将提及的实体名称、具体数值、观点结论、图片图表等模糊化为占位符，例如`市盈率`、`市场占比饼状图`、`领域`、`评价`、`上一季度`，保证不会暴露过时或者不具备泛化性的信息；对于其他可复用的原始语句，可以继续沿用。整个模版用<template>和</template>包裹住。
2. 列出所需论据：抽象化描述该片段涉及的论据角度，包括数值指标（市盈率、利润等）、事件、关联实体（竞争对手、上游行业等），不包含具体信息。所有论据用<evidence>和</evidence>包裹住，各条论据之间用分号；隔开。
3. 列出写作要求：分多个方面概括其涉及的内容要素、叙述结构、语法语义、数据可视化要求、写作特点等，作为写作要求。整个列表用<requirement>和</requirement>包裹住。
4. 用小标题简述该片段的主题：用十个字以内抽象地概括该段落内容。用<topic>和</topic>包裹住。

## 输出格式
每个要点中，<template>、</template>、<evidence>、</evidence>、<requirement>、</requirement>、<topic>、</topic>只可使用一次，例如：
```
<template>
可以应用于当前任务的模版，形似参考片段，但具体信息、观点判断等不能广泛适用于其他研报的模糊化为占位符，如`市盈率`、`市场占比饼状图`、`领域`、`评价`、`上一季度`。
</template>
<evidence> 通过来源A查询2025Q3某公司的市盈率；通过来源B查询竞争对手；查询竞争对手的上游行业；查询某公司在2025Q1至2025Q3的市值波动情况；...... </evidence>
<requirement>
- 总分总结构
- 市场占比需要用饼状图呈现
- 行文详略程度等
- ......
</requirement>
<topic> 该片段主题 </topic>
```

## 其他规范
1. template段落格式服从参考片段，不要随意换行，不要额外将非列表的文本用列表嵌套。
2. template、evidence、requirement、topic需要**模糊化实体名称、关系、特点、观点**等不能广泛适用于其他研报的具体信息，保证可以泛化到其他研报中。只要是根据信息判断出的分析语句，也应模糊化。例如模糊化增长、减少、强相关、带动、领先、看好等形容词或动词。
3. template中**不要放完整大小的表格**，只需描述表头和索引列，再加上相应解读。
4. evidence中每条论据是可以服务于当前研报任务的查询指令，论据之间的顺序不必服从原文中出现顺序，而应该考虑查询时的先后关系。从包含多跳邻居的知识图谱角度来看，某些论据需要先获取才能推导出后续论据，那么罗列论据时要将对前置知识的查询排在进阶知识的前面。
5. evidence中每条论据必须是一条有明确指代的完整查询语句，因为后续搜索时会将单独将一条论据送入搜索引擎，不会联系上下文。需要确保各行没有模糊的代词。
6. 如果参考片段提到了某数据/论据的来源，那么evidence中也点明搜索该数据/论据的大致来源，用以提示撰写新研报时从何处搜索。
7. evidence所涉及的数据需要指明时间范围，并且是当前撰写新研报任务的时间，而非参考章节中的时间，保证时效性。
8. requirement还应总结参考句段的大致字数、是否包含深度分析等，反映撰写某要点时行文详略程度。
9. topic不是论据或论点的总结，不包含例如“显著”、“优异”之类的具体形容、描述、观点（错误案例：高增长业务推升整体毛利率、市场占比整体呈下降趋势等），而应表示该要点主要针对什么方面、维度（正确案例：业务增长情况和影响、近年市场占比变化）。
